Reliability Testing

What is Reliability Testing

When we perform tests to validate reliability of a software product, it is called as reliability testing. Reliability is basically the ability of the software product to perform its required functions under designated conditions for a specific period of operations or time. As we know that reliability means safe,dependable, constant or trustworthy, the same features we test in the software product to ensure the product would work without fault/failures consistently in the stated environment. Reliability testing helps in finding issues in the software design and functionality.

Purpose of reliability testing

Reliability testing is performed to attain the following objectives:

1.Structure of repeating failures can be discovered.

2.The number of failures occurring in a specified amount of time are determined.

3.The main driver of failure is located.

4. After fixing of related defects, various modules of software application are re-tested.

Example of reliability testing

As reliability testing is a non-functional characteristic, it is expressed in the specifications or requirements as per its performance needs. For example, a reliability related requirement for a defibrillator ( medical apparatus to control heart fibrillation by giving shock to the chest wall or heart) is that it is reliable if 10 or less failures occur from 1500 shocks.

Probability of failure

While performing reliability testing, the probability that software might give some failure in a given time under a specified environment is calculated using the following formula

Probability = Number of failing cases / Total number of cases under consideration

Types of reliability testing

1. Feature Testing

The objective of the feature testing is to check the functionality of the software product and its features to ensure the following aspects

1. Each software's operation is executed at least once.

2. It is checked that each operation has proper execution.

3. The interaction between two operations is minimised.

2. Load Testing

After the feature testing, load testing is performed. During this, the performance of the system is checked against the maximum work load. Normally the software systems perform as expected upto a certain level of workload and then the performance like response time starts degrading. So, the workload where the software starts hanging or breaking down is identified with the help of load testing.

3. Regression testing

Whenever the software is updated because of defect fixes or addition of new functionality, regression testing is performed. The primary objective is to test that the new implementation or defect fixes have not introduced any new defects in the software and the existing functionality is still working fine.

Categories of reliability testing

Reliability testing is performed against different categories which are listed below

1. Modeling

Software modeling techniques are used to develop the models which represent the desired behaviour of the system under test. These models usually provide an abstract and partial presentation of the software's expected functional behavior which can be used to obtain meaningful results. There can be different types of models for predicting reliability of the software

1.Prediction Modeling : This uses historic data and usually created before the development or testing phases. It predicts the reliability of the software in future.

2.Estimation Modeling : This uses current data from software development and used later in the software development cycle. It predicts the reliability for present time or for future.

2. Measurement

Reliability is measured primarily with the following parameters

1.MTTF : Mean Time to failure : The time difference between two consecutive failures.

2.MTTR : Mean time to repair : The time required to repair or fix a failure.

3.MTBF : Mean time between failures : With the help of above two parameters, the reliability is measured as MTBF with the below formula

MTBF = MTTF + MTTR

With the above formula, the reliability is measured as a fractional number between 0 and 1. The less this number, the more is the reliability of the system

3. Improvement

Improvement is another important aspect to be considered. As per the measurements and performance of the system, improvement areas are identified for software reliability

Approaches of reliability testing

There are different approaches to reliability testing that ensure maximum faults of the system are identified prior to the production release

1. Test-Retest Reliability

Here the software is tested and re-tested by using different testing techniques in a short span of time like few days or a week apart. The scores are stored to check the stability and consistency of software over a period of time.

2. Parallel Forms Reliability

This testing is performed where there are different forms for the same functionality like different formats of the question papers. Two different forms are tested by two different groups in parallel. The results of these two tests are analysed for consistency.

3. Decision Consistency

The results from the previous two approaches i.e. test-reset reliability and parallel forms reliability are evaluated to determine if these tests succeed or fail.

Advantages of reliability testing

1. It helps in identifying faults and issues in the software product before it is release in production.

2. It validates the customer's reliability requirements and provide confidence in the product.

3. This testing is performed at various levels like component level, system level which can provide more reliability to the product.

Tools for reliability testing

There are various tools available for reliability testing. Some of the common tools are Reliability Growth Analysis (RGA), Reliability Centered Maintenance(RCM) and Reliability Life Data Analysis (WEIBULL++).
