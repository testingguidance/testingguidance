Test Design techniques

What are Test Design Techniques?

Test design techniques are used to identify test cases, test data and test conditions. There are different design techniques based on different parameters like type of system, contractual requirements, risks and development life cycle etc. The major ones are listed below

Black Box test design techniques

These specification based test design techniques derive test cases or test data based on test basis without taking the internal structure of the component or system under test into account. Some of the common characteristics are as below.

1. Models are used for the specification of software or components . These models may be formal or informal.

2. From these models, the test cases are derived.

There are numerous black box test design techniques , some of which are listed below. Please click on the individual items for details

1. Equivalence partitioning
2. Boundary value analysis
3. Decision table
4. State-transition testing
5. Use Case Testing

2. White Box test design techniques

These structure based techniques derive test cases or test data based on the internal structure of the component or system under test .Some of the common characteristics are as below.

1. Knowledge on how software is constructed like code and details of the design are used to derive test cases.

2. The test coverage is measured generally with the help of a tool. With the existing test cases, the coverage can be measured and increased with additional test cases.

There are numerous white box test design techniques , some of which are listed below. Please click on the individual items for details

1. Statement Testing and Coverage
2. Decision Testing and coverage
3. Condition Coverage
4. Modified condition decision coverage

3. Experience based techniques

As the name suggests, this test design technique is based on tester's experience. Here testers use their skills , intuition along with their experience on similar applications. These normally supplement the formal test designing techniques listed above and are effective in finding some defects which are less likely to be caught using the other techniques. Some examples are listed below.

1. Error guessing

This fault attack technique guesses the places where there are possible faults or errors and write tests around those areas on the basis of prior experience.

2. Exploratory Testing

Some of the important characteristics of exploratory testing are

1. It consists of concurrent test designing, test execution, test logging and learning.

2. It is carried out within time boxes.

3. Useful approach when specifications are incomplete. Also used when there is a lot of time pressure and most critical bugs need to be found.

4. It complements other formal testing techniques.
