Globalization Testing

What is Globalization Testing

In today's global context , where the virtual world knows no geographical constraints and needs from customers around the globe holds utmost importance, it is imperative to have the software products globally usable. The software should be able to perform well in the required locals and culture, also it should be able to handle all types of international inputs. The testing of these important aspects is Globalization testing. It is called as G11N because of 11 characters between G and N

Globalization testing covers Localization testing and Internationalization testing

These two types of testing go hand in hand. In localization, the product localization is done while taking the international standards into account. In internationalization, it is ensured that the product is usable for any region. Let's try to understand all these terms in details

Localization Testing

Localization is the process of customizing the global software for a specific locale(language, territory etc) or environment. The process involves translation of the software including icons, user manuals, documentation, graphics and other cultural specifications for the end users. For example when we want to search the web, google provides the facility to search in your native language like French, spanish, hindi etc. When we perform localization testing,we validate that application is working well with this localized version of the software. It is called as L10N because of 10 characters between L and N. The most important attributes validated in localization testing are as follows:

1. The main focus is on UI and content

2. Overall content is validated along with help text

3. Local laws and regulations are taken into account

4. Cultural sensitivity is considered during testing

5. Time and date formatting

6. Address and phone numbers formatting, postal code etc are tested

7. Tests related to currency handling are included if applicable

8. Paper sizes for printing

Internationalization Testing

Internationalization is the process of organizing the software code in a way that it is independent of any language, culture or regional information. Such software are tailored to keep all formatting functions which may need to be localised for example date and time formatting, address or phone number formatting are provided externally at run time so that basic functionality of the code or application is not broken. While performing internationalization testing, tester validates that the application is working uniformly with different regions and cultures. It is called as I18N because of 18 characters between I and N. The most important attributes validated in internationalization testing are as follows:

1. Functionality testing over different language environments using every type of possible international input

2. Language compatibility tests

3. UI validation, graphics content and localised text

4. Usability testing : Testing the ease of application use

5. Interoperability testing : Testing the application over cross operating systems, app versions etc

6. Installation Testing : Application is installed in different languages and validating the installation messages in different languages

Advantages of Globalization Testing

1. Support costs can be reduced as potential problems can be uncovered at an early stage.

2. Overall testing costs are reduced.

3. Overall testing time is reduced resulting in faster launch of products in global market.
