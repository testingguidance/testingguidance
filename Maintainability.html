Maintainability Testing

What is Maintainability Testing

When we perform tests to determine the maintainability of a software product, it is called as maintainability testing. Maintainability is basically the ease with which a software application or product can be updated to correct defects, changed to add new requirements, changed to make future requirements easier or can adapt to a modified environment. As the software should have the flexibility to change throughout the software life cycle specially in today's agile world , maintainability becomes a very important non functional characteristic.

Categories of maintainability testing

Product maintainability needs to be tested against various categories which are listed below:

1.Corrective maintenance: the system's maintainability is measured in terms of time taken in identifying and fixing defects in it.

2.Perfective maintenance: the system's maintainability is measured in terms of efforts required for system's modifications and improvements.

3.Adaptive maintenance: the system's maintainability is measured in terms of efforts required for adapting the changes in the environment.

4.Preventive maintenance: covers the actions for reducing future maintenance costs.

Attributes of maintainability testing

The following attributes are taken into account for maintainability:

1.Analyzability: This involves estimating the efforts for tracing defects along with the areas which require change in the system.

2.Changeability This is the software's ability to change.

3.Stability This is the software's ability to remain stable after undergoing changes.

4.Testability This is the testing effort required to validate the updated system.

5.Maintainability Compliance needs to be met as per the regulations.

Testing techniques for maintainability testing

Maintainability testing can be performed by applying either static testing techniques or dynamic testing techniques

1. Static Testing

For effective maintainability testing, static analysis and reviews should be included in the test plans. Some defects can be captured during code walk-through and inspection which might get missed during dynamic testing. The code structure and documentation of the software development process are checked in static testing with the help of inspection,reviews or walk-through

2. Dynamic Testing

Tests are executed to validate the different attributes mentioned in the above sections. After the software is modified for defect fixes, new functionality or changes in functionality, these changes are validated against conformance to requirements.

Advantages of maintainability testing

1. Every software product undergoes changes during development or post production against defect fixes or new functionality, maintainability testing ensure that the product would be able to function well post changes

2. The static testing used verify that algorithms are optimised, system is properly distributed, different standards for database approach, structured programming and user interfaces are properly followed. This helps in uncovering critical defects which help in maintaining and expanding the software in the long run.
