Security Testing

What is Security Testing

When we perform tests on a software to uncover vulnerabilities,threats,risks of the system and to find out that its data and resources are safe from possible intruders, then it is known as Security testing. In today's world, where everything is available online, most of the business transactions are performed online like online shopping, banking, stock trading , security testing has become absolute necessary to identify threats and measure vulnerabilities of the system . which can cause big losses like loss of reputation, revenue or confidential information. The security risks in the system are detected and fixed through coding by developers as a result of security testing. In the cyber-world, there are many hackers and pranksters who intend to break the system and may acquire confidential information so it is the need of the time to use applications and systems which we can trust upon. For desktop applications also, the security factor is really important.

Focus areas of Security testing

For web applications or websites, there are mainly four focus areas to be considered

1.Network Security: The network infrastructure like resources or policies might have vulnerabilities which can be uncovered .

2.Client-side application security: This is to ensure the manipulation of client browser or components can't be done.

3.Server-side application security: This is to ensure that server code and technologies are able to oppose any intrusions.

4. System-software security : The weaknesses in the application's software like operating system, database systems etc are assessed.

Let us take an example to understand it better. Through internet banking application, when we make an online payment, we can try to use the back arrows to see if still it displays the account information or any account details. Such simple tests can expose the security threats in our application.

Types and techniques of Security Testing

There are numerous types and techniques of security testing . The most important ones are discussed below

1. Vulnerability Scanning

The use of automated computer program to scan a system to identify security vulnerabilities in a network, is vulnerability scanning. This indicates the locations where a system can be threatened or exploited. There is no manual intervention or verification and the risk levels are set automatically through the tool.

2. Security Scanning

Security scanning is done with the help of manual or/and automated programs to identify potential security vulnerabilities in the network, operating system or web application. It also involves providing solutions for reducing the threats and risks.

3. Penetration testing

When we try to simulate the attack on the application from a malicious hacker, it is called as penetration testing. It helps in analysing potential vulnerabilities of a system from external hacking by gaining access to its information,data, functions and affecting its data integrity.

4. Ethical hacking

The hacking of the organization's software systems or network to expose security flaws in the system is ethical hacking. The hacker tries to search for vulnerabilities by bypassing the system security. These ethical hackers then suggest changes to the system to make them less likely to be penetrated by outside hackers

5. Risk Assessment

The security risks of the organization are analysed and classified as high, medium and low. This is done with the help of various interviews and discussions within the organization. Afterwards, controls and measures are recommended to reduce these risks

6. Security Auditing

It involves an internal inspection of the company's information system, applications and operating systems for finding security flaws. In this case, even line by line inspection of code can also be performed

7. Posture Assessment

This involves assessing the overall security structure of the organization by combining ethical hacking,security scanning and risk assessments.

8. SQL Injection

SQL injection attacks are the most common application attack techniques where malicious SQL statements are inserted into entry fields for execution. Doing this, the hacker or attacker can gain access to critical information from the server database. The input fields like text boxes, comments etc should be properly handled in the code for preventing SQL injections. This includes handling special characters or defining input length small to restrict input of malicious script.

9. Cross site scripting (XSS)

In Cross site scripting, the hackers or attackers inject client-side script into web pages viewed by different users and mislead the user into clicking on that URL. Once this code is executed by the other browser of the user, it can perform totally unexpected actions like gaining access to personal data, changing behaviour of the website etc. While performing security testing for this aspect, the tester need to check that any HTML or script is not accepted by the application. Cross site scripting testing is done for >, <, ' signs.

10. URL Manipulation

In the applications where HTTP GET method is used for information flow from client and server, hackers try to gain access to the important information by manipulating the website URL query strings which contain information in its parameters. In order to test it, the tester needs to modify a parameter value in the query string and check if the sever accepts it. Additionally, ad hoc testing can be done by testing random/encoded data which is included in requests or parameters.

Advantages of Security testing

1. The different attributes of security testing like Authentication,Authorization,Confidentiality,Availability,Integrity, Non-repudiation,Resilience indicates the advantages of security testing.

2. Security testing helps in gaining customer trust and avoids loss of revenue, downtime against security threats.
