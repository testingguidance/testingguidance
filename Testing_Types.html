Testing Types

There are numerous types of testing. There can be multiple ways to categorize the testing types based on following parameters.

1. Based on test purpose or objectives.

2. Based on degree of automation.

The important aspect is to understand these different types of testing with their purpose. The classification varies for different people as per their perspective.

1. Criteria: Test objectives or target of testing

1. Functional Testing

Testing of "What the system does" is functional testing. Basically the expected functional behaviour of a system, sub system or a component is verified. Also, the functional tests take interoperability between systems into account.These functions or features can be found in requirements specification, use cases or functional specification or may be even undocumented. It can be applied at all test levels. Main items are listed in the figure on the right.

Different Types of Functional Testing
  • Black Box Testing
  • Security Testing
  • Interoperability Testing
  • Globalization Testing
  • Smoke and Sanity Testing

    • 2. Non Functional Testing

    Testing of "How the system works" is non functional testing. The non functional tests aim to measure characteristics of software and systems which are quantified on varying scale. These tests aim to verify that the performance requirements of the system are working as expected . In this testing , the external behaviour of the system is verified. Main items are summarized in the figure on the right.

    Different types of Non Functional Testing
  • Performance Testing
  • Load Testing
  • Stress Testing
  • Usability Testing
  • Maintainability Testing
  • Portability Testing
  • Reliability Testing
  • Black Box Testing

    • 3. Structural Testing

    Testing of "Software structure/Architecture" is structural testing. The test coverage of a code structure is assessed to measure the thoroughness of testing. Generally it is expressed as a percentage of the items being covered. If the results show that 100% tests are not covered, more tests can be designed to cover missing items thus increasing the coverage. Main items of structural testing are summarized in the figure on the right.

    Different types of Structural Testing
  • White Box Testing

    • 4. Testing related to changes

    This includes confirmation testing when defects in the software are fixed and changes require re-testing in order to confirm that the defect has been removed. Also, when the software or environment is changed, new defects may be encountered in the existing software , these changes are tested with the help of regression testing . Essentially , the same set of tests are run again to ensure the modification has not impacted the existing software. Main items are summarized in the figure on the right.

    Different types of change related testing
  • Confirmation testing (or Re-testing)
  • Regression testing

    2. Criteria: Degree of Automation

    1. Manual Testing

    The process of performing testing activities manually without the help of automation tools is called manual testing. It helps in finding defects in the system and is the most traditional way of testing. All types of testing can be performed manually. Normally any system, sub system or component is tested manually in the first iteration and then can be automated for next iterations. Main items are listed in the figure on the right.

    2. Automation Testing

    The process of performing testing activities with the help of automation tools is called automation testing. All types of testing can be performed using automation. In automation testing, both test case execution and comparison of expected and actual results are done through automation tools being used. Please refer the section Automation testing for details. Main items are listed in the figure on the right

    3. Semi automated testing

    The process of performing testing activities manually along with automation tools. Sometimes it is not possible to automate the complete test case, a part of the test case can be automated and the rest is verified manually by the tester. Often automation tool is involved in the former part to produce test data which can be used by the tester manually in the latter part. Main items are summarized in the figure on the right.
